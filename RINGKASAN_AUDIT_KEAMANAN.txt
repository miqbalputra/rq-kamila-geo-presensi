🔴 KERENTANAN KRITIS (5) - HARUS DIPERBAIKI!
═══════════════════════════════════════════════════════════════

1. CORS Terlalu Permisif
   ❌ Mengizinkan akses dari SEMUA domain
   ❌ Siapapun bisa akses API dari website lain
   
2. Tidak Ada Autentikasi di API
   ❌ API bisa diakses tanpa login
   ❌ Siapapun bisa hapus/edit data
   
3. SQL Injection di Filter
   ❌ Parameter tidak divalidasi dengan baik
   ❌ Potensi SQL injection
   
4. Password Tidak Divalidasi Kekuatannya
   ❌ Bisa set password "123"
   ❌ Mudah di-brute force
   
5. Database Credentials Hardcoded
   ❌ Password database di file PHP
   ❌ Jika file bocor, database bisa diakses


⚠️ KERENTANAN MEDIUM (12)
═══════════════════════════════════════════════════════════════

6.  Tidak ada rate limiting (brute force attack)
7.  Tidak ada input validation (XSS attack)
8.  Error messages terlalu detail (info leakage)
9.  HTTPS tidak di-enforce (data bisa di-sniff)
10. Session tidak secure (session hijacking)
11. Tidak ada CSRF protection
12. Tidak ada security logging
13. Tidak ada file upload validation
14. Tidak ada backup strategy
15. Tidak ada API versioning
16. Tidak ada monitoring
17. Geolocation tidak divalidasi


✅ YANG SUDAH BAIK (8)
═══════════════════════════════════════════════════════════════

✓ Password hashing dengan bcrypt
✓ PDO dengan error mode exception
✓ Session management
✓ JSON response format konsisten
✓ Charset UTF-8

✓ Separation of concerns
